Priority Queue — Stage-Advancing Tasks

Each open task is ranked by priority = order value × (1 + days overdue). Click any row to expand — you'll see a pre-written instruction generated from the order's data, plus controls to send it to the right person. Owner-only tasks (card or bank access) are locked to the card holder; everything else is delegable.

Tasks update live when stages change. When you tick a trigger or set a manual stage on an order, that order's task switches to the new stage's task automatically (e.g. an order moved from Stage 4 to Stage 5 drops post_return_label and picks up contact_retailer_investigation).

Missing Information Queue

Tasks already get ranked by £ × (1 + days_overdue). That ranking is meaningless when a task can't even start because a field is null — ranking a stuck task high doesn't unstick it. This queue reroutes attention to the blocker. Chase the missing field; the queue starts flowing again.

Information-priority formula
info_priority = £value × (1 + days_missing) × blocking_factor

Method Choice

Where the recovery method is decided per order. The method auto-resolves once all the variables the tree needs are present — orders split into auto-decided and missing-info-to-decide. Use the explorer to test variables and commit a method, or set BBM. Time awaiting decision is tracked per order and surfaced as a bottleneck on the dashboard.

Delivery

The two questions this answers: when's my order arriving and when's the courier coming to collect the return. Awaiting delivery and awaiting collection at a glance — tap a card to open the order, or mark it received.

Stores

Set up stores first — before placing orders. A store's return logic and contacts determine what every order from it will need, so completing the store record up front means each new order auto-resolves its method instead of stalling. Every store from order history is here; order tracking links by store name, and courier is auto-derived from past shipments.

Parcelshop

Drop-off shops near your base postcode, seeded from courier APIs. Returns posted here link back to the order; success rate is computed from real returns. Add notes to any shop.
